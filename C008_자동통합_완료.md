## 🎯 자동 트리거 조건

V2 검증이 **자동으로** 실행되는 경우:

### **1. 환경변수 설정**
```bash
ENABLE_V2_AUTO_TRIGGER=true
```
→ 항상 V2 실행

### **2. 분석 깊이가 높을 때**
```typescript
analysisDepth: 'CORE' 또는 'FULL'
```
→ BASIC은 안 하고, CORE/FULL만 V2 실행

### **3. HOVCS 점수가 애매할 때**
```
70% ~ 90% 사이
```
→ 추가 검증 필요해서 V2 자동 실행

### **4. 위반 사항이 있을 때**
```
violations.length > 0
```
→ 문제 있으니까 V2로 더 깊게 분석

### **5. 코드 파일 검증할 때**
```typescript
targetPath: './src/myCode.ts'
```
→ .ts, .js, .py 등 실제 코드 파일이면 V2 실행

## 🔍 V2 검증 단계별 설명

자동 트리거되면 다음 7단계 검증:

### **1. SPECIFICATION (명세)**
- 요구사항 추출
- 제약조건 식별
- 성능 예산 확인

### **2. MODULARIZATION (모듈화)**
- 모듈 구조 분석
- 의존성 그래프 생성
- 영향 범위 계산

### **3. GENERATION (생성)**
- 코드 + 테스트 검증
- 테스트 커버리지 확인

### **4. EXECUTION (실행)**
- **실시간 시뮬레이션** 🔥
- TypeError 예측
- Null pointer 탐지
- 성능 추정

### **5. DEBUGGING (디버깅)**
- 오류 자동 수정 제안
- (실패 시에만 실행)

### **6. REGRESSION (회귀)**
- 기존 기능 영향도 확인
- 성능 회귀 체크

### **7. DEPLOYMENT (배포)**
- 3단계 품질 게이트
  - HIGH: 아키텍처, 보안
  - MEDIUM: 인터페이스, 성능
  - LOW: 테스트, 코드 품질
