### **Stage 간 데이터 흐름**

```
┌─────────────────────────────────────────────────────────────┐
│ Stage 1: SPECIFICATION                                       │

│ - 성능 예산 정의 여부 확인                                   │

                ▼

│ Output: functions[], imports[], exports[], smells[]          │
└───────────────┬─────────────────────────────────────────────┘
                │ functions[] 재사용
                ▼
┌─────────────────────────────────────────────────────────────┐
│ Stage 3: TEST_SCAFFOLDING                                    │
│ - Stage 2의 functions[] 재사용                               │
│ - 테스트되지 않은 함수 식별                                  │
│ - 자동 테스트 템플릿 생성                                    │
│ Output: testTemplate, uncoveredFunctions[]                   │
└───────────────┬─────────────────────────────────────────────┘
                │ complexity 데이터 재사용
                ▼
┌─────────────────────────────────────────────────────────────┐
│ Stage 4: PERFORMANCE_BUDGET                                  │
│ - Stage 1의 performanceBudget 검증                          │
│ - Cyclomatic Complexity 상세 측정                           │
│ - Heavy Patterns 탐지 (5가지)                                │
│ - Nesting Depth 측정                                         │
│ Output: perfIssues[], optimization recommendations           │
└───────────────┬─────────────────────────────────────────────┘
                │ 모든 결과 종합
                ▼
┌─────────────────────────────────────────────────────────────┐
│ Stage 5: DEPLOYMENT                                          │
│ - Stage 1,3의 testCoverage 최종 검증                        │
│ - 보안 패턴 탐지 (Secrets, SQL, eval)                       │
│ - Stage 2의 Critical Path 최종 확인                         │
│ Output: deploymentReadiness, finalRecommendations            │
└───────────────┬─────────────────────────────────────────────┘
                │ 전체 통합
                ▼
            Overall Score
        (모든 stage 평균 + 가중치)
```

### **HOVCS 2.0과의 통합**

```typescript

async applyEngineering(request: VibeComplianceRequest): Promise<ComplianceReport> {
  // 1. HOVCS 2.0 기본 검증 (6 phases)

  // 2. V2 Enhanced 자동 트리거

    v2ValidationPipeline: v2Pipeline,

}
```

3. ✅ **HOVCS score: 70-90%** → Borderline 케이스
4. ✅ **violations.length > 0** → 위반 사항 있을 때
