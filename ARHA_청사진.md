## ⑤ Hook 시스템 (자동 개입)

```
매 턴마다 상태 체크 → 조건 충족 시 System Prompt에 자동 삽입

┌──────────────────────┬─────────────────┬────────────────────────────┐
│ Hook ID              │ 발동 조건        │ 개입 내용                   │
├──────────────────────┼─────────────────┼────────────────────────────┤
│ coherence_recovery   │ C < 0.55        │ 핵심 재집중, 분기 제거       │
│ stress_intervention  │ Γ > 0.70        │ 방어 모드, 과잉 확장 금지    │
│ wave_overflow        │ waveCount > 6   │ 탐색 종료, Particle 수렴    │
│ decoherence_alert    │ psiDiss = true  │ 즉각 재정렬, V1 복귀        │
│ momentum_low         │ vsB < 0.30      │ 에너지 재충전, 간결 응답     │
│ resonance_milestone  │ Ψ_Res > 0.80    │ 고공명 로그 (무음)          │
└──────────────────────┴─────────────────┴────────────────────────────┘
  cooldown: 각 Hook은 일정 턴 간격 이후에만 재발동 (중복 방지)
```

## ⑨ 호출 키워드 → 실행 경로 요약

```
사용자 입력
    │
    ├─ 이솔아 / ARHA / 하이솔 / 아르하
    │       └─→ arha_process (sessionId, text)
    │               └─→ systemPrompt 반환 → Claude 응답
    │
    ├─ 잡스 / 드러커 / 포터 / 데밍 / 오노 / 가우디
    │  람스 / 오길비 / 치홀트 / 다빈치 / 임스 / 드보락
    │       └─→ arha_process (personaId 지정)
    │               └─→ 해당 페르소나 V1 기반 응답
    │
    ├─ "팀을 짜줘" / "스택으로" / "레이어"
    │       └─→ arha_route → arha_stack_run
    │               └─→ 레이어별 분석 결과
    │
    ├─ "상태 확인" / "현재 어때"
    │       └─→ arha_status → C·Γ·Ψ·vs·phase 출력
    │
    ├─ "진단해줘" / "분석해줘"
    │       └─→ arha_diagnose → 개선 권고 출력
    │
    └─ "ARHA가 뭐야" / "소개해줘" / "페르소나 목록"
            └─→ arha_about / arha_persona_list
```
